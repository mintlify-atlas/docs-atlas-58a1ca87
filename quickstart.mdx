## Installation

Install the library using Go modules:

```bash
go get github.com/h22rana/jsonlogic2sql@latest
```

## Basic usage

<Steps>
  <Step title="Import the library">
    ```go
    import "github.com/h22rana/jsonlogic2sql"
    ```
  </Step>

  <Step title="Convert JSON Logic to SQL">
    Use the `Transpile` function with a dialect and JSON Logic expression:

    ```go
    sql, err := jsonlogic2sql.Transpile(
        jsonlogic2sql.DialectBigQuery,
        `{">": [{"var": "amount"}, 1000]}`,
    )
    if err != nil {
        panic(err)
    }
    fmt.Println(sql) // Output: WHERE amount > 1000
    ```
  </Step>

  <Step title="Use in your queries">
    The generated SQL can be used directly in your database queries:

    ```go
    query := fmt.Sprintf("SELECT * FROM orders %s", sql)
    // SELECT * FROM orders WHERE amount > 1000
    ```
  </Step>
</Steps>

## Common patterns

### Logical operations

Combine multiple conditions with `and` and `or`:

<CodeGroup>
```go AND conditions
sql, err := jsonlogic2sql.Transpile(
    jsonlogic2sql.DialectBigQuery,
    `{"and": [
        {"==": [{"var": "status"}, "pending"]},
        {">": [{"var": "amount"}, 5000]}
    ]}`,
)
// Output: WHERE (status = 'pending' AND amount > 5000)
```

```go OR conditions
sql, err := jsonlogic2sql.Transpile(
    jsonlogic2sql.DialectBigQuery,
    `{"or": [
        {">=": [{"var": "failedAttempts"}, 5]},
        {"in": [{"var": "country"}, ["CN", "RU"]]}
    ]}`,
)
// Output: WHERE (failedAttempts >= 5 OR country IN ('CN', 'RU'))
```
</CodeGroup>

### Nested conditions

Build complex nested logic:

```go
sql, err := jsonlogic2sql.Transpile(
    jsonlogic2sql.DialectBigQuery,
    `{"and": [
        {">": [{"var": "transaction.amount"}, 10000]},
        {"or": [
            {"==": [{"var": "user.verified"}, false]},
            {"<": [{"var": "user.accountAgeDays"}, 7]}
        ]}
    ]}`,
)
// Output: WHERE (transaction.amount > 10000 AND (user.verified = FALSE OR user.accountAgeDays < 7))
```

### Conditional logic with if

Use the `if` operator for conditional expressions:

```go
sql, err := jsonlogic2sql.Transpile(
    jsonlogic2sql.DialectBigQuery,
    `{"if": [{">": [{"var": "age"}, 18]}, "adult", "minor"]}`,
)
// Output: WHERE CASE WHEN age > 18 THEN 'adult' ELSE 'minor' END
```

### Working with arrays

Check if values exist in arrays:

```go
sql, err := jsonlogic2sql.Transpile(
    jsonlogic2sql.DialectBigQuery,
    `{"in": [{"var": "country"}, ["US", "CA", "MX"]]}`,
)
// Output: WHERE country IN ('US', 'CA', 'MX')
```

## Using a transpiler instance

For advanced usage, create a `Transpiler` instance to configure options and reuse across multiple transpilations:

<Steps>
  <Step title="Create a transpiler">
    ```go
    transpiler, err := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectBigQuery)
    if err != nil {
        panic(err)
    }
    ```
  </Step>

  <Step title="Transpile from JSON string">
    ```go
    sql, err := transpiler.Transpile(`{">": [{"var": "amount"}, 1000]}`)
    ```
  </Step>

  <Step title="Transpile from pre-parsed map">
    ```go
    logic := map[string]interface{}{
        "or": []interface{}{
            map[string]interface{}{">=": []interface{}{map[string]interface{}{"var": "failedAttempts"}, 5}},
            map[string]interface{}{"in": []interface{}{map[string]interface{}{"var": "country"}, []interface{}{"CN", "RU"}}},
        },
    }
    sql, err := transpiler.TranspileFromMap(logic)
    // Output: WHERE (failedAttempts >= 5 OR country IN ('CN', 'RU'))
    ```
  </Step>
</Steps>

## Getting conditions without WHERE

Use `TranspileCondition` when you need just the condition without the `WHERE` keyword:

```go
condition, err := jsonlogic2sql.TranspileCondition(
    jsonlogic2sql.DialectBigQuery,
    `{">": [{"var": "amount"}, 1000]}`,
)
// condition = "amount > 1000"

// Use in a custom query
query := fmt.Sprintf("SELECT * FROM orders WHERE %s AND created_at > '2024-01-01'", condition)
```

## Choosing a SQL dialect

The library supports multiple SQL dialects. Specify the dialect when creating a transpiler:

<CodeGroup>
```go BigQuery
transpiler, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectBigQuery)
```

```go PostgreSQL
transpiler, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectPostgreSQL)
```

```go Spanner
transpiler, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectSpanner)
```

```go DuckDB
transpiler, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectDuckDB)
```

```go ClickHouse
transpiler, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectClickHouse)
```
</CodeGroup>

Different dialects may generate different SQL for the same JSON Logic expression. See the [SQL dialects](/dialects) guide for details.
