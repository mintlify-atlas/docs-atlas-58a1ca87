The jsonlogic2sql library provides a comprehensive API for converting JSON Logic expressions to SQL WHERE clauses across multiple SQL dialects.

## Core components

### Quick start functions

For simple use cases, use the package-level convenience functions:

- [Transpile()](/api/functions#transpile) - Convert JSON Logic to SQL WHERE clause
- [TranspileCondition()](/api/functions#transpilecondition) - Convert to SQL condition without WHERE
- [TranspileFromMap()](/api/functions#transpilefrommap) - Convert from parsed JSON

### Transpiler instance

For advanced use cases with custom operators or schema validation:

- [Transpiler](/api/transpiler) - Main transpiler instance with full configuration
- [NewTranspiler()](/api/functions#newtranspiler) - Create basic transpiler
- [NewTranspilerWithConfig()](/api/functions#newtranspilerwithconfig) - Create with configuration

### Type definitions

Core types for configuration and customization:

- [TranspilerConfig](/api/types#transpilerconfig) - Configuration options
- [Dialect](/api/types#dialect) - SQL dialect constants
- [OperatorHandler](/api/types#operatorhandler) - Custom operator interface
- [OperatorRegistry](/api/types#operatorregistry) - Custom operator management

### Schema validation

Optional schema support for field validation:

- [Schema](/api/schema#schema) - Field collection and validation
- [FieldSchema](/api/schema#fieldschema) - Individual field definition
- [FieldType](/api/schema#fieldtype) - Field type constants

### Error handling

Structured error types for robust error handling:

- [TranspileError](/api/errors#transpileerror) - Structured error with code and path
- [ErrorCode](/api/errors#errorcode) - Error code constants
- [AsTranspileError()](/api/errors#astranspileerror) - Extract error from chain
- [IsErrorCode()](/api/errors#iserrorcode) - Check specific error code

## Usage patterns

### Simple transpilation

```go
import "github.com/h22rana/jsonlogic2sql"

sql, err := jsonlogic2sql.Transpile(
    jsonlogic2sql.DialectBigQuery,
    `{"==": [{"var": "status"}, "active"]}`,
)
// Result: WHERE status = 'active'
```

### Transpiler with schema

```go
schema := jsonlogic2sql.NewSchema([]jsonlogic2sql.FieldSchema{
    {Name: "amount", Type: jsonlogic2sql.FieldTypeNumber},
    {Name: "status", Type: jsonlogic2sql.FieldTypeEnum, AllowedValues: []string{"active", "inactive"}},
})

config := &jsonlogic2sql.TranspilerConfig{
    Dialect: jsonlogic2sql.DialectBigQuery,
    Schema:  schema,
}

transpiler, err := jsonlogic2sql.NewTranspilerWithConfig(config)
```

### Custom operators

```go
transpiler, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectBigQuery)

transpiler.RegisterOperatorFunc("length", func(op string, args []interface{}) (string, error) {
    if len(args) != 1 {
        return "", fmt.Errorf("length requires exactly 1 argument")
    }
    return fmt.Sprintf("LENGTH(%s)", args[0]), nil
})

sql, _ := transpiler.Transpile(`{"length": [{"var": "email"}]}`)
// Result: WHERE LENGTH(email)
```

### Error handling

```go
sql, err := transpiler.Transpile(jsonLogic)
if err != nil {
    if tpErr, ok := jsonlogic2sql.AsTranspileError(err); ok {
        fmt.Printf("Error code: %s\n", tpErr.Code)
        fmt.Printf("Path: %s\n", tpErr.Path)
        fmt.Printf("Operator: %s\n", tpErr.Operator)
    }
}
```
