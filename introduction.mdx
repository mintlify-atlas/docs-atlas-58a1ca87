## Overview

jsonlogic2sql is a Go library that converts [JSON Logic](http://jsonlogic.com/) expressions into SQL. It provides a clean, type-safe API for transforming JSON Logic rules into SQL WHERE clauses or standalone conditions, with support for multiple SQL dialects.

<CodeGroup>
```go Simple example
package main

import (
    "fmt"
    "github.com/h22rana/jsonlogic2sql"
)

func main() {
    sql, err := jsonlogic2sql.Transpile(
        jsonlogic2sql.DialectBigQuery,
        `{">": [{"var": "amount"}, 1000]}`,
    )
    if err != nil {
        panic(err)
    }
    fmt.Println(sql) // Output: WHERE amount > 1000
}
```

```go Complex conditions
sql, err := jsonlogic2sql.Transpile(
    jsonlogic2sql.DialectBigQuery,
    `{"and": [
        {"==": [{"var": "status"}, "pending"]},
        {">": [{"var": "amount"}, 5000]}
    ]}`,
)
// Output: WHERE (status = 'pending' AND amount > 5000)
```
</CodeGroup>

## Supported operators

| Category | Operators |
|----------|----------|
| **Data access** | `var`, `missing`, `missing_some` |
| **Comparison** | `==`, `===`, `!=`, `!==`, `>`, `>=`, `<`, `<=` |
| **Logical** | `and`, `or`, `!`, `!!`, `if` |
| **Numeric** | `+`, `-`, `*`, `/`, `%`, `max`, `min` |
| **Array** | `in`, `map`, `filter`, `reduce`, `all`, `some`, `none`, `merge` |
| **String** | `in`, `cat`, `substr` |

## Supported dialects

| Dialect | Constant |
|---------|----------|
| Google BigQuery | `DialectBigQuery` |
| Google Cloud Spanner | `DialectSpanner` |
| PostgreSQL | `DialectPostgreSQL` |
| DuckDB | `DialectDuckDB` |
| ClickHouse | `DialectClickHouse` |

## Important notes

<Warning>
**SQL injection:** This library does NOT handle SQL injection prevention. The caller is responsible for validating input and using parameterized queries where appropriate.
</Warning>

<Note>
**Semantic correctness:** This library assumes that the input JSON Logic is semantically correct. The transpiler generates SQL that directly corresponds to the JSON Logic structure without validating the logical correctness of the expressions.
</Note>
