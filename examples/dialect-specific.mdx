Different SQL dialects have unique syntax for certain operations. This guide shows real examples of how jsonlogic2sql adapts to each dialect.

## Supported dialects

jsonlogic2sql supports five SQL dialects:

- **BigQuery** - Google BigQuery
- **Spanner** - Google Cloud Spanner
- **PostgreSQL** - PostgreSQL database
- **DuckDB** - DuckDB analytics database
- **ClickHouse** - ClickHouse OLAP database

```go
import "github.com/h22rana/jsonlogic2sql"

// Create transpiler for specific dialect
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectBigQuery)
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectPostgreSQL)
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectSpanner)
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectDuckDB)
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectClickHouse)
```

## Common operations (same across dialects)

Many operations produce identical SQL across all dialects:

### Comparisons

```json
{">": [{"var": "amount"}, 1000]}
```

All dialects produce:
```sql
WHERE amount > 1000
```

### Logical operators

```json
{
  "and": [
    {"==": [{"var": "status"}, "active"]},
    {">": [{"var": "score"}, 100]}
  ]
}
```

All dialects produce:
```sql
WHERE (status = 'active' AND score > 100)
```

### NULL checks

```json
{"==": [{"var": "deleted_at"}, null]}
```

All dialects produce:
```sql
WHERE deleted_at IS NULL
```

### Arithmetic

```json
{"+": [{"var": "price"}, {"var": "tax"}]}
```

All dialects produce:
```sql
WHERE (price + tax)
```

### String concatenation

```json
{"cat": [{"var": "firstName"}, " ", {"var": "lastName"}]}
```

All dialects produce:
```sql
WHERE CONCAT(firstName, ' ', lastName)
```

### CASE statements

```json
{
  "if": [
    {">": [{"var": "age"}, 18]},
    "adult",
    "minor"
  ]
}
```

All dialects produce:
```sql
WHERE CASE WHEN age > 18 THEN 'adult' ELSE 'minor' END
```

## Choosing a dialect

Select the dialect that matches your target database:

```go
import "github.com/h22rana/jsonlogic2sql"

// For Google BigQuery
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectBigQuery)

// For Google Cloud Spanner
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectSpanner)

// For PostgreSQL
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectPostgreSQL)

// For DuckDB
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectDuckDB)

// For ClickHouse
tr, _ := jsonlogic2sql.NewTranspiler(jsonlogic2sql.DialectClickHouse)
```

## Dialect compatibility

All dialects support:
- ✅ Data access (var, missing, missing_some)
- ✅ Comparisons (equality, inequality, greater/less than)
- ✅ Logical operators (and, or, not, if)
- ✅ Arithmetic (addition, subtraction, multiplication, division, modulo, max, min)
- ✅ String operations (cat, substr, in)
- ✅ Array operations (map, filter, reduce, all, some, none, merge, in)
